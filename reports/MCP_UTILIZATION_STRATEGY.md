## 🔍 Detailed Assessment

### 1. SQLite MCP (The Operational Backbone)
- **Đánh giá:** Công cụ thiết yếu nhất. Thay thế các script Python đơn lẻ để truy vấn SQLite.
- **Tác động:** Giảm 90% thời gian viết code phụ trợ chỉ để xem dữ liệu.
- **Rủi ro:** Cần đảm bảo path tới DB chính xác trong config.

### 2. Memory MCP (The Long-term Cognition)
- **Đánh giá:** Biến các "Active Thoughts" tĩnh thành Knowledge Graph động.
- **Tác động:** Giúp Agent "nhớ" các sai lầm trong quá khứ (`FAILED_PATHS.json`) mà không cần tốn token đọc lại toàn bộ file.
- **Chiến lược:** Mỗi khi kết thúc Task, các Key Findings phải được đẩy vào Memory MCP.

### 3. Sequential Thinking (The Logical Guardrail)
- **Đánh giá:** Ngăn chặn việc nhảy cóc logic hoặc ảo giác khi xử lý các hàm phức tạp.
- **Tác động:** Tăng độ tin cậy của các quyết định kiến trúc (Architectural Decisions).

### 4. Playwright MCP (The Visual Eyes)
- **Đánh giá:** Mandatory cho mọi thay đổi liên quan đến `.html` hoặc UI framework.
- **Tác động:** Đảm bảo layout không bị vỡ (Self-Reflect QA).

## 📜 Updated Workflow Rules
1. **[Data First]**: Trước khi sửa code liên quan đến Data/Database, BẮT BUỘC dùng SQLite MCP để audit cấu trúc hiện tại.
2. **[Memory Anchor]**: Đầu mỗi Task, dùng Memory MCP `read_graph` để lấy bối cảnh. Cuối Task dùng `add_observations` để lưu tri thức mới.
3. **[Visual QA]**: Mọi sửa đổi UI phải kèm theo screenshot từ Playwright MCP.
4. **[Deep Reasoning]**: Các task gắn tag `[ARCHITECTURE]` hoặc `[REFACTOR]` phải khởi động Sequential Thinking.
