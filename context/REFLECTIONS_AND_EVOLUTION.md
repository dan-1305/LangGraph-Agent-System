| **Sự cố Docker VHDX Nuốt Ổ Cứng (14GB):** Quá trình build và chạy các AI Container rác khiến file ổ đĩa ảo của Docker (WSL2) phình to lên 14GB, làm sập toàn bộ hệ thống lưu trữ của máy tính. | Lỗi Quản trị Hạ tầng (Infrastructure Mismanagement): User cài Docker nhưng thiếu kiến thức về cơ chế cấp phát ổ đĩa động của WSL2 (chỉ phình to không tự co lại) và lười dọn dẹp dangling images. | **1.** Thay đổi quy trình CI/CD: Bổ sung lệnh `docker system prune -a --volumes` vào kịch bản dọn dẹp định kỳ.<br>**2.** Đưa lệnh `diskpart` (compact vdisk) vào cẩm nang sinh tồn phần cứng để định kỳ giải phóng ổ C. |

| **Chiến thuật "3-Tier Backup" (Giải pháp tối thượng cho file `.env`):** Lỗi NTFS Lock gây crash khiến Admin yêu cầu một giải pháp chia nhỏ file để chống phá hoại thay vì dùng quyền OS cứng nhắc.
