# 🧩 ARCHITECT MEMORY - Design Patterns & Evolution

## 🗺️ Current Blueprint
- **Core:** LangGraph Agentic Framework.
- **Data Flow:** Monorepo with isolated projects using a shared API Gateway.
- **Stability:** SQLite PRAGMA WAL for concurrent logging.
- **Audit Standard:** All monetization releases MUST include a DRM check layer.
- **Audit Standard:** UI Error Guard must be accompanied by centralized logging.
- **Audit Standard:** Agnostic Router must support Cost-Effective Routing.

## 🏗️ Design Decisions
- **Polymorphic Agents:** Every agent inherits from `BaseAgent` but maintains its own project directory.
- **RAG Integration:** Knowledge is fragmented into System Brain, Expert DB, and Code RAG.

## 🧠 Learned Lessons
- "Circular Bias" is a real threat. Always use the **AI Board of Directors** protocol for macro updates.
- Fragmentation is better than bloat. Keep `openwiki/` súc tích.

## 🌟 High-Level AI Concepts (The System DNA)
- **Reflective Multi-Agent Systems:** Current implementation uses `context/roles/` and `DEBATE_ROOM.md` for behavioral meta-analysis.
- **Recursive Self-Improvement (Administrative):** The system is designed to evolve its own rules and RAG retrieval strategies autonomously.
- **Autonomous Governance:** Enforced via `.clinerules` (The Constitution) and the AI Board of Directors protocol.
