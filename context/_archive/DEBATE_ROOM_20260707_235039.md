## 📅 BOARD DEBATE: 2026-07-07 21:00 (GODOT 4 REPACK FINALIZATION)
**Topic:** Triển khai cơ chế `--txt-to-bin` để xử lý triệt để lỗi Repack Godot 4.

### 🎭 Participants:
1.  **[CEO Cline]**: Orchestrator.
2.  **[System Architect]**: Infrastructure Lead.
3.  **[Dev]**: Senior Backend Developer.
4.  **[QA]**: Reliability Guard.
5.  **[Security]**: Cyber Security Lead.

### 🗣️ Debate Log:
- **[CEO]**: Chúng ta cần hoàn tất `godot_translator` để thương mại hóa. Lỗi `Parse Error: Expected '['` trên Godot 4 là rào cản cuối cùng. Dev hãy trình bày giải pháp.
- **[Dev]**: Tôi sẽ sửa `repack()` trong `pack_manager.py`. Quy trình mới: Quét toàn bộ `.tscn` đã dịch -> Chạy `gdre_tools --txt-to-bin` -> Xóa `.tscn` gốc và các file `.remap` -> Pack lại PCK. Điều này ép game phải đọc file nhị phân `.scn` tương ứng.
- **[Architect]**: Giải pháp này bẻ gãy cơ chế "Remap" của Godot nhưng lại là cách duy nhất để qua mặt Engine bản Release không có trình phân tích Text. Cần lưu ý đường dẫn `gdre_tools.exe` phải linh hoạt.
- **[QA]**: Rủi ro là nếu `txt-to-bin` thất bại, game sẽ mất Scene đó. Tôi đề xuất thêm bước Verify: Chỉ xóa `.tscn` nếu `.scn` đã được tạo thành công. Ngoài ra cần test trên game "The Last Node".
- **[Security]**: Việc xóa file `.remap` là an toàn vì chúng ta đang cung cấp file nhị phân thực thụ thay vì điều hướng. Cần đảm bảo không có Path Traversal khi quét file.
- **[Dev]**: Tôi sẽ dùng `Path.rglob` để quét an toàn và thêm logic Verify của QA.

### ⚖️ Final Decision (The Consensus):
- Phê duyệt triển khai cơ chế convert `.tscn` sang `.scn` tự động trong `PackManager`.
- Bắt buộc Verify sự tồn tại của file `.scn` trước khi xóa file nguồn.
- Thực hiện test lâm sàng trên "The Last Node" ngay sau khi code xong.
